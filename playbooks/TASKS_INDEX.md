# Reusable Task Index

This folder turns the long website-building thread into reusable task artifacts.

For each major task category, there are two files:

- a `playbooks/*.md` file: the working rules, standards, checklists, and decisions
- a `prompts/*.prompt.md` file: a ready-to-paste prompt for future sessions

## Task Categories

1. `01-website-build-system.md`
   Covers the whole personal-site build: positioning, homepage, projects, services, about/career narrative, blog integration, and overall consistency.

2. `02-blog-editorial-system.md`
   Covers how blog posts should be written and edited across the site: tone, citations, references, structure, figure treatment, and polish expectations.

3. `03-blog-ui-social-deployment.md`
   Covers article UX, table of contents behavior, social cards, metadata, preview behavior, and deployment workflow.

4. `04-linkedin-positioning-assets.md`
   Covers LinkedIn-facing decisions: profile positioning, Featured link copy, posting strategy, cover-image logic, and profile-photo guidance.

## Suggested Usage

- If the goal is to continue or rebuild the website, start with:
  `playbooks/01-website-build-system.md`
- If the goal is to write or revise technical articles, start with:
  `playbooks/02-blog-editorial-system.md`
- If the goal is to improve previews, sharing, or live publishing, start with:
  `playbooks/03-blog-ui-social-deployment.md`
- If the goal is to improve LinkedIn presentation and personal branding, start with:
  `playbooks/04-linkedin-positioning-assets.md`

The matching prompts in `prompts/` are designed to be pasted directly into a future AI session.
